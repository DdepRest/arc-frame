# USER_GUIDE.html redesign — спецификация

**Файл результата:** `docs/USER_GUIDE.html`
**Дата:** 2026-06-28
**Версия продукта:** 3.38.0
**Статус:** Draft (interview завершён, до имплементации)

---

## 1. Цель

Капитальный редизайн одностраничного HTML-мануала по программе **A.R.C. Frame** (калькулятор КП для москитных сеток). Цель — превратить статическую справку в **интерактивный онбординг** для новых сотрудников, где ключевые функции программы эмулируются прямо в браузере.

---

## 2. Целевая аудитория

**Новые сотрудники** — люди, только пришедшие в компанию. Нужно объяснять контекст, терминологию и логику расчётов, а не только показывать кнопки. Язык: **русский**.

---

## 3. Масштаб

**Капитальный редизайн** — допускается полная переработка структуры, стилей, контента и интерактивности. Текущий файл считается устаревшим.

---

## 4. Области улучшения (все 4 аспекта одновременно)

- **Визуал и стиль** — новая палитра, типографика, отступы, тени, иконки, иллюстрации.
- **Контент и структура** — переписать тексты, переразбить секции, изменить порядок по принципу «сначала что делать → потом что внутри».
- **Интерактивность** — кликабельные элементы, эмуляция функций программы, анимации.
- **Техническое качество** — accessibility (WCAG 2.1 AA), семантическая разметка, валидный HTML/CSS, перформанс.

---

## 5. Архитектурные ограничения

| Ограничение | Решение |
|-------------|---------|
| Один файл или много | **Один HTML-файл** (всё inline) |
| Картинки реальных экранов | **Нет** — только CSS/SVG-эмуляция |
| Языки | **Только русский** (без переключателя) |
| Печатная версия | **Нет** — только для экрана |
| Зависимости | **Vanilla HTML/CSS/JS** — без сторонних библиотек |

Файл должен работать при открытии двойным кликом `file://` без сервера.

---

## 6. Визуальный стиль

**Modern Corporate** — чистый дизайн в духе Fluent UI / Material Design:

- **Палитра:** белый фон, **синий акцент** (#2563eb), нейтральные серые рамки и тени. Без неона, без glassmorphism.
- **Типографика:** системный sans-serif, чёткая иерархия (h1 крупный и тяжёлый, body 15–16 px, captions 12–13 px), математические формулы — моноширинный.
- **Формы:** прямоугольные карточки с тонкими рамками и `border-radius` 12–16 px.
- **Иконки:** SVG inline (не эмодзи), 1.5 px stroke.
- **Тени:** мягкие, естественные (`0 2/_ 8/_ rgba(15,23,42,0.06)`), не глубокие.
- **Иллюстрации окна программы:** CSS-миниатюры интерфейса (div-ы с правильными пропорциями и цветами), интерактивные.

## 7. Интерактивные функции (обязательные)

### 7.1. Anwis pill-bar (эмуляция segmented control)

В секции Anwis пользователь может **кликать по пилюлям ББ 60 / ББ 70 / ПП / Проём / Габарит**. При клике:

- Активная пилюля получает `background: white`, тень, синий текст.
- Под pill-bar появляется **живое объяснение выбранного режима** с формулой и примером:
  - Ввод: 1000 × 1000
  - Расчёт: 1002 × 970 (для ББ 60)
  - На завод: 980 × 950 (расчёт −20 мм по обеим осям)
- В таблице ниже подсвечивается соответствующая строка.
- Значения формул должны точно совпадать с реальной логикой из `AnwisSize.ОтВвода` (см. `Models/AnwisSize.cs`).

### 7.2. Mini-калькулятор стоимости

Отдельный **интерактивный блок** в секции Workflow или рядом с Products. Поля:

- Тип продукта (выбор из 9).
- Цвет (фильтруется по типу).
- Anwis-режим (только для Anwis; по умолчанию ББ 60).
- Ширина, мм (`<input type="number">`).
- Высота, мм (`<input type="number">`).
- Количество (`<input type="number">`).

Поля ниже пересчитываются автоматически при изменении любого входа:

- **Площадь / длина** (по формуле продукта).
- **Расчётный размер для Anwis** (с применением формулы режима).
- **Стоимость**, руб. = Площадь × Цена (из захардкоженного прайса) × Кол-во.

Цены можно взять из `prices.json`:
- Anwis Белый = 1800, Коричневый = 1900
- На навесах Белый = 2900, Коричневый = 3000
- Оконная на метал. крепл. Белый = 3200, Коричневый = 3300
- Отлив / Козырёк / Короб — 2150 (Белый/Коричневый/Антрацит), 2650 (Золотой дуб)
- Уплотнение = 250 (Серый / Чёрный)

Продукты без площади (ПСУЛ, Откос материал, Работа, Брус, Пояс, Доставка) — рассчитываются по `Сумма = Кол-во × Цена`. Цена = 100 (ПСУЛ), 0 (Откос материал, Работа — показать «как ввести вручную»).

Используемый JS должен быть ванильным (`addEventListener` + `input` events, без библиотек).

### 7.3. Product preview (карточки продуктов)

Клик по карточке продукта (не по формуле) **выделяет её рамкой** и:

- Подсвечивает карточку (рамка 2 px синяя, фон чуть светлее).
- Показывает в side-панели (или под grid): развёрнутую информацию о продукте — для каких задач используется, типовые размеры, ограничения.
- В mini-калькуляторе автоматически подставляет выбранный продукт.

Двойной клик или Esc — снимает выделение.

## 8. Трекинг прогресса

**Linear progress bar** в верхней части страницы (sticky, под или над навигацией). Считает процент прочитанного по секциям:

- При первом появлении секции в viewport — увеличивается счётчик.
- Прогресс-бар заполняется слева направо с процентом и числом (например, «3 / 7 разделов»).
- При достижении 100 % показывает поздравительный toast: «Готово! Вы прочитали всю инструкцию».

Хранится в `localStorage` — при следующем визите пользователь видит, где остановился.

## 9. Структура секций

1. **Hero** — заголовок «Калькулятор КП для москитных сеток», подзаголовок, CTA «Начать» (scroll вниз), линейный прогресс.
2. **Workflow** — 6 шагов. Шаг 6 включает кнопки-эмуляции «Сохранить», «На завод», «Печать».
3. **Mini-калькулятор** — интерактивная форма расчёта (см. 7.2). Цель: за 30 секунд новый сотрудник получает первый КП.
4. **Продукты** — 9 карточек (см. 7.3), с подсветкой при клике.
5. **Anwis** — пилюли (см. 7.1) + таблица 5 режимов.
6. **Горячие клавиши** — 4 реальные (Ctrl+S, Ctrl+P, Enter, Esc) с теми же physical-key стилем.
7. **Автообновление** — компактная секция (без моков 다운로드).
8. **Footer** — версия, ссылка на GitHub, год.

## 10. Технические требования

### 10.1. HTML
- Валидный HTML5 (`<!DOCTYPE html>`).
- Семантические теги: `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`.
- У каждой секции — `id` для anchor-ссылок.
- Все интерактивные элементы доступны с клавиатуры (Tab / Enter / Esc).

### 10.2. CSS
- Мобильная адаптивность: breakpoints 900px и 560px.
- Без сторонних шрифтов (системный стек).
- CSS-переменные в `:root` для палитры.
- Комментарии по основным секциям.

### 10.3. JS (vanilla)
- IIFE-обёртка для изоляции.
- Никакого `eval`, никаких внешних запросов.
- `localStorage` для прогресса чтения и последнего выбранного режима Anwis.
- `IntersectionObserver` для анимации появления и подсчёта прочитанных секций.
- Без зависимостей.

### 10.4. Accessibility (WCAG 2.1 AA)
- Контраст текста ≥ 4.5:1.
- Все кнопки имеют `aria-label` или видимый текст.
- Фокусное состояние (`:focus-visible`) видимо.
- `prefers-reduced-motion` отключает анимации.

### 10.5. Performance
- Inline JS — один файл, без зависимостей. Целевой размер < 80 КБ (HTML+CSS+JS).
- `<meta name="viewport">` для мобильных.
- `defer` для inline script или размещение в конце `<body>`.
