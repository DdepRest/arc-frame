# CALCULATION_TEST_CASES.md

## Назначение

Этот файл хранит эталонные примеры расчётов.  
AI должен сверяться с ними перед любыми изменениями в расчётной логике.  
Если код после изменения выдаёт другие результаты — это регрессия.

**Статусы:**
- `baseline из кода` — значение взято из текущих юнит-тестов, не подтверждено владельцем вручную.
- `подтверждено владельцем` — владелец проверил на калькуляторе/в программе и подтвердил.

## Case 11 — КП: размеры и суммы

| Параметр | Значение |
|----------|----------|
| Товар | Anwis, ББ 60 |
| Входная (сырая) ширина | 1000 |
| Входная (сырая) высота | 1000 |
| Количество | 1 |
| Цена | 1800 |

### Ожидаемые значения в КП (по текущему коду)

| Показатель | Значение | Статус |
|-----------|----------|--------|
| Размеры в КП | **1002 × 970 мм** | baseline из кода |
| CalculatedValue в КП | 0.972 м² | baseline из кода |
| Total в КП | 1749.60 руб. | baseline из кода |
| Сумма прописью | «одна тысяча семьсот сорок девять рублей 60 копеек» | требует подтверждения владельца |

**Важно:** В КП попадают **хранимые (расчётные)** размеры (`OrderItem.Width`/`Height`) — то есть 1002×970 для ББ 60. Это текущее поведение `PrintService.cs` (строки 48–49: `item.Width:F0`, `item.Height:F0`).

> **Требует подтверждения владельца:** Должны ли в КП показываться **введённые** размеры (1000×1000) вместо **расчётных** (1002×970)? Если да — это изменение поведения, требующее согласования.
