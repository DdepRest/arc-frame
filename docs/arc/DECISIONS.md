# DECISIONS.md

## Важные принятые решения

### 1. Стек: C# + WPF + .NET 8

**Решение:** Десктопное приложение на WPF, а не веб или кроссплатформа.

**Почему:**
- Приложение работает на Windows в офисе/на объекте.
- Нужна печать и работа с файловой системой.
- Владелец не программист — простота поддержки важнее кроссплатформенности.

---

### 2. Single-file publish

**Решение:** Сборка в один .exe файл (`PublishSingleFile=true`).

**Почему:** Простота распространения — один файл для скачивания и запуска.

**Последствия:**
- `Assembly.GetName().Version` не работает (null в single-file).
- Используем `AssemblyInformationalVersionAttribute` для версии.

---

### 3. Хранение данных в %AppData%

**Решение:** Все пользовательские данные (заказы, настройки, цены) хранятся в `%AppData%\MosquitoNetCalculator\`, а не рядом с .exe.

**Почему:** При автообновлении .exe заменяется — данные должны сохраниться.

**Когда принято:** v3.28.0.

---

### 4. Автообновление через GitHub Releases + watchdog .bat

**Решение:** Собственный механизм обновления вместо Velopack (который был в v3.28-v3.34).

**Почему:** Velopack Flow вызвал технические проблемы (конфликт дельта-пакетов, необходимость повторных публикаций).

**Как работает:**
1. Скачивается ZIP с GitHub Releases.
2. Проверяется SHA-256.
3. Запускается `watchdog.bat`, который ждёт выхода приложения.
4. `.bat` распаковывает ZIP, заменяет `.exe`, запускает новую версию.
5. При неудаче — откат к `.exe.bak`.

**Когда принято:** v3.34.4+ (после ухода от Velopack).

---

### 5. Partial-классы для разбиения больших файлов

**Решение:** `MainWindow.xaml.cs` разбит на 5 partial-файлов (`Items`, `Orders`, `TitleDirty`, `WindowChrome`, `DataGrid`). `OrderItem.cs` разбит на `Calculations` и `Installation`.

**Почему:** Улучшение читаемости и навигации. Каждый файл отвечает за свою область.

**Когда принято:** v3.34.4.

---

### 6. Три слоя размеров Anwis

**Решение:** `AnwisSize` хранит 3 представления: Ввод (пользовательский), Расчёт (для цены), Завод (для производства).

**Почему:** Для разных целей нужны разные размеры. Клиенту показываем одно, заводу — другое.

**Когда принято:** v3.26.0 (переработка Anwis).

---

### 7. CalculatedValue и Total НЕ сохраняются в JSON

**Решение:** В `OrderItemData` (DTO) хранятся только исходные данные (Name, Color, Width, Height, Quantity, Price). CalculatedValue и Total всегда пересчитываются при загрузке.

**Почему:** Избежать рассинхронизации при изменении цен или формул.

**Когда принято:** v3.22.0.

---

### 8. Единый источник версии — .csproj

**Решение:** Версия задаётся только в `<Version>X.Y.Z</Version>` в `.csproj`.

**Почему:** Единое место правки. Всё остальное (AssemblyVersion, FileVersion, InformationalVersion) генерируется автоматически .NET SDK.

**Когда принято:** С самого начала, усилено в v3.34.5.

---

### 9. Миграции цен в PriceService

**Решение:** При загрузке prices.json применяются миграции: переименования, обновление цен, удаление устаревших цветов, добавление новых товаров.

**Почему:** Пользователи могут иметь старый prices.json от предыдущих версий. Нужно поддерживать совместимость.

**Когда принято:** Постепенно, последняя консолидация в v3.22+.

---

### 10. Тёмная тема через DynamicResource

**Решение:** Все цвета — через `DynamicResource` в `Brushes.xaml`. Переключение темы = замена словаря ресурсов.

**Почему:** Мгновенное переключение без перезапуска.

**Последствия:**
- Frozen кисти в Style/ControlTemplate нельзя анимировать — нужна осторожность.
- `ThemeService` пересоздаёт кисти вместо их модификации.

---

### 11. HTML-шаблон для КП (не FlowDocument)

**Решение:** КП генерируется как HTML из `print_template.html`, а не как WPF FlowDocument.

**Почему:**
- Лучший контроль над вёрсткой.
- SVG-чертежи для каждого товара.
- Печать через WebView2 даёт PDF через диалог печати Windows.

---

### 12. Юнит-тесты рядом с проектом

**Решение:** Отдельный проект `MosquitoNetCalculator.Tests` с `InternalsVisibleTo`.

**Почему:** Тесты могут проверять `internal` методы (например, `UpdateService.ParseSafe`).

**Текущее покрытие:**
- Расчёты, Anwis-размеры, монтаж.
- Печать (template text checks).
- Завод (текстовые сценарии).
- Хранение (roundtrip JSON).
- Обновления (парсинг версий).
- Цены (миграции).
- UI-диалоги (manual checklist).
