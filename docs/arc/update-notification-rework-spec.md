# Update Notification System Rework — Spec

> **Статус:** Draft  
> **Дата:** 2026-06-26
> **Last verified:** 2026-07-12 — документ перепроверен в рамках Фазы 3 рефакторинга; спецификация остаётся в статусе Draft, технические изменения не начаты. Статус Draft и затронутые модули актуальны.  
> **Затронутые модули:** `UpdateService`, `ActionBarControl`, `TitleBarControl`, `DialogService`, `UpdateLog`, `ToastService`, `UpdatesTabControl`

---

## 1. Проблема

Текущая система оповещений об обновлениях имеет несколько UX-проблем:

1. **Наложение UI-элементов.** `DownloadProgressPanel` (правая часть ActionBar) залезает на `DirtyIndicator` ("Есть изменения") при узком окне — оба живут в одном `Grid`-строке.
2. **Toast-спам.** При проверке обновлений показывается 3–4 toast'а подряд: "Проверка наличия обновлений...", "У вас последняя версия", "Установка обновления..." и т.д.
3. **Диалог без контекста.** `ShowUpdateAvailable` показывает только номер версии ("Версия 3.36.2") без changelog — пользователь не понимает, что нового.
4. **Неочевидная точка входа.** Toast при запуске говорит "Нажмите «Проверить обновления» в Настройках" — пользователь должен найти gear → context menu → пункт. Не интуитивно.
5. **Прогресс в ActionBar.** Компактный прогресс-бар в кнопочной полосе визуально чужероден и занимает место.

---

## 2. Решение — обзор

| Аспект | Было | Станет |
|--------|------|--------|
| Уведомление при запуске | Toast "Доступна новая версия... Нажмите в Настройках" | Автоматический диалог с changelog + кнопка "Обновить" |
| Диалог скачивания | Модальный, только версия, без changelog | Диалог с changelog + прогресс скачивания внутри |
| Прогресс скачивания | `DownloadProgressPanel` в ActionBar (правая часть) | Тонкая полоска (2–3px) под TitleBar |
| Toast-сообщения | 4–5 toast'ов в процессе | Только ошибки + "Проверка целостности" + "Установка обновления" |
| Отклонённое обновление | Забывается до ручной проверки | Показывается снова при каждом запуске |
| Закрытие диалога при скачивании | N/A (диалог закрывался до скачивания) | Скачивание продолжается в фоне (полоска в TitleBar) |
| Красная точка на Настройках | Есть | Остаётся как напоминание |

---

## 3. Детальное поведение

### 3.1. Проверка при запуске (`CheckOnStartupAsync`)

**Текущий flow:**
```
Startup → fetch manifest → toast "Доступна новая версия X.Y.Z" → сохранить pending version
```

**Новый flow:**
```
Startup → fetch manifest → если версия новее:
  → показать диалог ShowUpdateAvailable() (автоматически)
  → если "Скачать" → начать скачивание (совмещённый диалог+прогресс)
  → если "Отмена" → закрыть диалог, сохранить pending version, показать UpdateBadge
```

- **Убрать** toast "Доступна новая версия: X.Y.Z. Нажмите «Проверить обновления»..."
- **Убрать** toast "Проверка наличия обновлений..." (при автоматической проверке)
- **Оставить** toast "Проверка наличия обновлений..." при **ручной** проверке (из меню Настроек)
- **Убрать** toast "У вас последняя версия" — заменить на toast "Обновлений нет ✓" (Success) при ручной проверке

### 3.2. Диалог `ShowUpdateAvailable` — редизайн

Диалог должен показывать:

1. **Шапка:** "Доступно обновление" + badge с номером версии
2. **Changelog:** Список изменений **всех пропущенных версий** (от `CurrentVersion + 1` до `Latest`):
   - Данные берутся из `update-log.json` (embedded resource, уже есть в проекте)
   - Формат: карточки версий (как в `UpdatesTabControl`) — компактный вариант
   - Если changelog слишком длинный — ограничить высоту + скролл
3. **Кнопки:** "Отмена" + "Скачать и установить"
4. **Прогресс скачивания:** После нажатия "Скачать":
   - Кнопки "Отмена"/"Скачать" заменяются на прогресс-бар + процент
   - Кнопка ✕ в title bar диалога **остаётся активна** — закрытие НЕ отменяет загрузку
   - После закрытия диалога прогресс виден в TitleBar-полоске
   - После завершения скачивания → автоматически продолжить flow (SHA-256 → watchdog → restart)

### 3.3. Прогресс скачивания в TitleBar

**Расположение:** Тонкая полоска (2–3px) под TitleBar (над контентом окна).

**Поведение:**
- Полоска видна **только** во время активного скачивания (`IsDownloading == true`)
- Цвет: `Accent` (из текущей темы)
- Анимация: плавное заполнение слева направо (как VS Code / Chrome)
- Не показывает процент или текст — только визуальный прогресс
- При `IsDownloading == false` — полоска скрывается с анимацией fade-out

**Реализация:**
- Добавить `ProgressBar` в `MainWindow.xaml` (или `TitleBarControl.xaml`) между TitleBar и основным контентом
- Стиль: `Height="3"`, без рамки, `BorderThickness="0"`, `Background="Transparent"`, `Foreground="{DynamicResource Accent}"`
- Подписка на `UpdateService.ProgressChanged` (уже существует)

### 3.4. Toast-спам — сокращение

| Toast | Действие |
|-------|----------|
| "Проверка наличия обновлений..." (авто) | **Убрать** |
| "Проверка наличия обновлений..." (ручная) | **Оставить** |
| "Доступна новая версия: X.Y.Z..." | **Убрать** (заменено диалогом) |
| "У вас последняя версия" | **Заменить** на "Обновлений нет ✓" (Success) |
| "Не удалось получить список обновлений" | **Оставить** (ошибка) |
| "Не удалось определить версию обновления" | **Оставить** (ошибка) |
| "Проверка целостности архива..." | **Оставить** (важный статус) |
| "Установка обновления..." | **Оставить** (важный статус) |
| "Не удалось скачать обновление:" (MessageBox) | **Заменить** на toast Error + закрыть диалог |
| "Хеш-сумма не совпадает" (MessageBox) | **Заменить** на toast Error + закрыть диалог |
| "Проверка обновлений уже выполняется..." | **Оставить** |

### 3.5. Отклонённое обновление — повтор при каждом запуске

**Текущий flow:**
- `CheckOnStartupAsync` → toast → `SavePendingUpdateVersion(latest)` → UpdateBadge
- При следующем запуске → снова toast → и т.д.

**Новый flow:**
- `CheckOnStartupAsync` → диалог → если "Отмена" → `SavePendingUpdateVersion(latest)` → UpdateBadge
- При следующем запуске → снова диалог (не toast!)
- Пользователь видит диалог **каждый запуск**, пока не обновится

**Убрать** логику "один раз показал toast, потом только точка" — теперь это всегда диалог.

### 3.6. Закрытие диалога во время скачивания

- Кнопка ✕ в title bar диалога **активна** во время скачивания
- Закрытие диалога **НЕ отменяет** загрузку — скачивание продолжается в фоне
- Прогресс виден в TitleBar-полоске
- По завершении скачивания → SHA-256 → watchdog → restart (как обычно)

### 3.7. Ручная проверка из меню Настроек

**Текущий flow:**
- "Проверить обновления" → `CheckAndApplyAsync` → toast'ы → диалог → скачивание

**Новый flow:**
- "Проверить обновления" → toast "Проверка наличия обновлений..." → `CheckAndApplyAsync`
- Если есть обновление → тот же диалог с changelog + прогресс
- Если нет → toast "Обновлений нет ✓" (Success)
- Ошибка → toast Error

## 4. Технические изменения

### 4.1. `UpdateService.cs`

- `CheckOnStartupAsync()`: вместо `ToastService.ShowToast(...)` → вызывать диалог `DialogService.ShowUpdateAvailable()` на UI thread
- Передавать `MainWindow` как owner (нужен доступ к window из startup flow)
- Добавить параметр `isAutomatic = true` в `CheckAndApplyAsync` для различения авто/ручной проверки
- Убрать toast "Проверка наличия обновлений..." при `isAutomatic == true`
- Заменить MessageBox-ошибки на toast Error при `isAutomatic == true`

### 4.2. `DialogService.cs` — `ShowUpdateAvailable()`

- Принимать `IEnumerable<UpdateItem>` (changelog для пропущенных версий)
- Показывать changelog в ScrollViewer (макс. высота ~200px)
- После нажатия "Скачать" — заменить кнопки на ProgressBar + процент
- Не закрывать автоматически после завершения скачивания
- Вернуть `bool` (скачать/отмена) как сейчас, но логика скачивания переносится в диалог

**Альтернатива:** Оставить `ShowUpdateAvailable` как чистый confirmation, а прогресс вынести в отдельный `ShowDownloadProgress` — зависит от архитектурного предпочтения.

### 4.3. `MainWindow.xaml` — TitleBar-полоска

Добавить между TitleBar и основным контентом:

```xml
<!-- Update download progress — thin bar under title bar -->
<ProgressBar x:Name="UpdateDownloadBar"
             Grid.Row="0"  <!-- или нужный Row -->
             Height="3"
             Minimum="0" Maximum="100"
             Value="{Binding ...}"
             Visibility="Collapsed"
             BorderThickness="0"
             Background="Transparent"
             Foreground="{DynamicResource Accent}"
             VerticalAlignment="Bottom"/>
```

Подписка на `UpdateService.ProgressChanged` в `MainWindow.xaml.cs`:
```csharp
UpdateService.ProgressChanged += (_, _) =>
{
    UpdateDownloadBar.Visibility = UpdateService.IsDownloading
        ? Visibility.Visible
        : Visibility.Collapsed;
    UpdateDownloadBar.Value = UpdateService.DownloadProgress;
};
```

### 4.4. `ActionBarControl.xaml` — убрать DownloadProgressPanel

- Удалить `DownloadProgressPanel` из XAML (Border + ProgressBar + TextBlock)
- Удалить `OnDownloadProgressChanged` handler из code-behind
- Удалить подписку `UpdateService.ProgressChanged += OnDownloadProgressChanged`
- `UpdateBadge` (красная точка) — **оставить**

### 4.5. `UpdateLog.cs` — получение changelog для пропущенных версий

Добавить метод:
```csharp
/// <summary>
/// Возвращает UpdateItem[] для всех версий строго newerThan.
/// Используется для показа changelog в диалоге обновления.
/// </summary>
public static UpdateItem[] GetChangesSince(Version currentVersion)
```

- Читает из `_entries` (уже загруженный `update-log.json`)
- Фильтрует версии > `currentVersion`
- Возвращает в порядке от старой к новой

## 5. Что НЕ меняется

- `UpdateService.CheckAndApplyAsync` — основной flow (fetch → confirm → download → SHA → watchdog → restart) остаётся
- `WatchdogService` — не трогать
- `UpdateManifest`, `UpdateItem` модели — не трогать
- `releases.json` формат — не трогать
- `update-log.json` формат — не трогать
- `UpdatesTabControl` (вкладка "История обновлений") — не трогать
- `AppSettingsService` (pending version storage) — не трогать

7. **Toast-замены** — убрать/заменить toast'ы по таблице §3.4
8. **Тесты** — обновить `UpdateServiceTests`, `UpdateLogTests`

## 7. Edge Cases

| Сценарий | Поведение |
|----------|-----------|
| Нет интернета при запуске | `FetchManifestAsync` вернёт null → молча выйти, никаких toast'ов |
| Медленный интернет | Progress bar в TitleBar показывает %, диалог открыт |
| Закрытие диалога во время скачивания | Скачивание продолжается в фоне (TitleBar-полоска) |
| Ошибка скачивания | Toast Error + закрыть диалог + полоска исчезает |
| Ошибка SHA-256 | Toast Error + удалить temp ZIP |
| Пользователь уже на последней версии (ручная проверка) | Toast "Обновлений нет ✓" (Success) |
| Пользователь уже на последней версии (авто) | Молча выйти |
| `update-log.json` не содержит пропущенных версий | Показать только badge версии без changelog (fallback) |
| Диалог открыт + пользователь закрывает окно приложения | `Owner` relationship обеспечит закрытие диалога |
| Несколько запусков подряд с отклонением | Диалог показывается каждый раз (§3.5) |
