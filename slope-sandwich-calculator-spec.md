# Slope Sandwich Calculator — Specification

## Version
Draft 1 — 2026-07-06

## Summary

Реализовать **автоматический калькулятор откосов из сэндвич-панелей** для ПВХ-конструкций (окна, балконные блоки). Пользователь вводит 3 размера (высота, ширина, глубина), система автоматически рассчитывает 8 позиций материалов и работы, а в КП попадают только 2 итоговые строки: «Откос» (сумма материалов) и «Работа за откос» (сумма работы).

---

## 1. Входные данные

### 1.1 Размеры (на одну конструкцию/окно)

| Поле | Единицы | Пример | Примечание |
|------|---------|--------|------------|
| **Высота (H)** | мм | 1500 | Как в существующих полях QuickAdd |
| **Ширина (W)** | мм | 700 | Как в существующих полях QuickAdd |
| **Глубина (D)** | авто (мм/м) | 150 или 0.15 | Автоопределение: если начинается с `0.` → метры; иначе > 10 → мм; с подсказкой `(150 мм → 0.15 м)` |

**Правило автоопределения глубины:**
- Если строка начинается с `0.` → парсим как метры (напр. `0.15` → 0.15 м)
- Иначе: целое число → мм (напр. `150` → 0.15 м, `10` → 10 мм)
- В UI рядом с полем отображается подсказка: `= 0.15 м` (результат конвертации) — чтобы пользователь видел, что не ошибся

### 1.2 Количество окон (Q)

Берётся из поля **«Кол-во»** в QuickAdd (как для любого товара). Qty = количество окон/конструкций. Все материалы и работа умножаются на Q.

## 2. 8 позиций расчёта (на 1 окно, Q=1)

### 2.1 Вспомогательные величины

```
P3 = (W + 2×H) / 1000   — периметр трёх сторон (верх + 2 бока), метры
P4 = 2 × (W + H) / 1000  — периметр четырёх сторон, метры
S  = P3 × D              — площадь поверхности откосов, м²
Area = W × H / 1_000_000 — площадь окна, м² (для определения типа)
```

### 2.2 Таблица позиций

| # | Позиция | Формула (на 1 окно) | Ед. | Цена | Источник цены |
|---|---------|---------------------|-----|------|---------------|
| 1 | **Сэндвич** | `P3 × D × Price` | руб. | 1200 ₽/м² | Вкладка «Цены»: `Сэндвич` |
| 2 | **Пена** | `1 × Price` | баллон | 750 ₽ | Вкладка «Цены»: `Пена (откос)` |
| 3 | **Герметик** | `ceil(N / 4) × Price` | тюбик | 350 ₽ | Вкладка «Цены»: `Герметик (откос)` |
| 4 | **Скотч** | `ceil(N / 3) × Price` | моток | 135 ₽ | Вкладка «Цены»: `Скотч (откос)` |
| 5 | **Старт** | `strips × Price` | полоса 3 м | 135 ₽ | Вкладка «Цены»: `Старт (откос)` |
| 6 | **F-планка** | `strips × Price` | полоса 3 м | 250 ₽ | Вкладка «Цены»: `F-планка (откос)` |
| 7 | **Пеноплекс** | `sheets × Price` | лист | 450 ₽ | Вкладка «Цены»: `Пеноплекс (откос)` |
| 8 | **Работа** | `P4 × Price` | руб. | 600 ₽/м.п. | Вкладка «Цены»: `Работа за откос` |

Где **N = общее количество откосов в заказе** (сумма Q по всем окнам).

### 2.3 Расчёт полос Старта и F-планки (оптимизация раскроя)

Дано: 3 отрезка — `[W, H, H]` (в мм). Полоса = 3000 мм. Задача: нарезать 3 отрезка из минимального числа полос.

**Правила:**
- Остаток от полосы можно использовать для другого отрезка, **если он полностью покрывает этот отрезок**
- Нельзя соединять два остатка (напр. 800+800 для 1500 — нельзя)
- Остаток, меньший любого из оставшихся отрезков, идёт в утиль

**Алгоритм (greedy):**
```
pieces = sort_descending([W, H, H])  // от большего к меньшему
offcuts = []   // доступные остатки
strips = 0

for piece in pieces:
    // ищем наименьший подходящий остаток
    best = min(offcuts.filter(o ≥ piece))
    if best exists:
        offcuts.remove(best)
        if best - piece > 0:
            offcuts.add(best - piece)
    else:
        strips += 1
        remainder = 3000 - piece
        if remainder > 0:
            offcuts.add(remainder)

return strips
```

**Примеры:**
- W=700, H=1500 → pieces=[1500,1500,700]: 1-я полоса(1500, ост.1500)→2-й отр.(1500 из ост.)→3-й отр.(новая полоса) → **2 полосы**
- W=1500, H=700 → pieces=[1500,700,700]: 1-я(1500, ост.1500)→2-й(700 из ост., ост.800)→3-й(700 из ост.800, ост.100) → **1 полоса**
- W=1000, H=2100 → pieces=[2100,2100,1000]: 1-я(2100,ост.900)→2-я(новая,ост.900)→3-я(новая, ост.2000) → **3 полосы**

### 2.4 Расчёт листов Пеноплекса

Определяется **автоматически по площади окна**:

| Площадь окна (Area) | Тип окна | Листов пеноплекса |
|---------------------|----------|-------------------|
| ≤ 2.0 м² | 1–2 створки | **1 лист** |
| 2.0–3.5 м² | 3 створки | **2 листа** |
| > 3.5 м² | Балконный блок | **3 листа** |

Пороги основаны на стандартных размерах ПВХ-окон (ГОСТ 23166-99):
- 1-ств.: до ~900×1500 (≤1.35 м²)
- 2-ств.: до ~1500×1500 (≤2.25 м²)
- 3-ств.: до ~2100×1500 (≤3.15 м²)
- Балконный блок: ~2100×2200 (≥3.78 м²)

### 2.5 Экономия (герметик и скотч)

Экономия считается **по всему заказу** (всем откосам вместе):

```
N_total = сумма Quantity всех «Откос» в заказе
Герметик (тюбиков) = ceil(N_total / 4)  // 1 тюбик на 4 откоса
Скотч (мотков)     = ceil(N_total / 3)  // 1 моток на 3 откоса
```

**Важно:** При добавлении нового откоса или изменении Q существующего, ВСЕ строки «Откос» в заказе должны пересчитать герметик и скотч (т.к. N_total изменился).

## 3. Итоговые строки в КП

В DataGrid (и в печатном КП) отображаются **ровно 2 строки**:

| № | Название | Ед. | Сумма (на 1 окно) | Сумма (всего) |
|---|----------|-----|-------------------|---------------|
| 1 | **Откос** | шт. | Σ(пп.1–7) = Сэндвич+Пена+Герметик+Скотч+Старт+F+Пеноплекс | × Q |
| 2 | **Работа за откос** | шт. | П.8 = P4 × 600 | × Q |

Обе строки — `ManualPieceProducts`, `CalculatedValue=1`. Это позволяет пользователю редактировать итоговую цену вручную при необходимости.

### 3.1 Поведение строк в DataGrid

- **«Откос»** — как существующий «Откос материал»: `IsManualPiece`, `WidthOnly` (ширина = глубина откоса для справки). Ширина/Высота из DataGrid не влияют на расчёт (подтягиваются из SlopeCalculation).
- **«Работа за откос»** — `IsManualPiece`, без Width/Height.
- Обе строки: при выделении → в боковой панели детализации показывается расшифровка.

- «Откос» появляется в выпадающем списке типов (между существующими)

### 4.4 Авто-пересчёт

- При **изменении размеров** (H/W/D) в панели → немедленный пересчёт всех 8 позиций

- При **ручном изменении** кол-ва/цены в детализации → пересчёт итогов

## 5. Модель данных

### 5.1 SlopeCalculation (новый класс)

```csharp
public class SlopeCalculation
{
    // Входные параметры
    public double WidthMm { get; set; }        // Ширина окна, мм
    public double HeightMm { get; set; }       // Высота окна, мм
    public double DepthM { get; set; }         // Глубина откоса, метры
    
    // 8 позиций (каждая — количество + цена за единицу + сумма)
    public SlopeMaterial Sandwich { get; set; }   // Сэндвич (м² × ₽/м²)
    public SlopeMaterial Foam { get; set; }       // Пена (шт × ₽)
    public SlopeMaterial Sealant { get; set; }    // Герметик (шт × ₽)
    public SlopeMaterial Tape { get; set; }       // Скотч (шт × ₽)
    public SlopeMaterial StartProfile { get; set; } // Старт (шт × ₽)
    public SlopeMaterial FProfile { get; set; }   // F-планка (шт × ₽)
    public SlopeMaterial Penoplex { get; set; }   // Пеноплекс (шт × ₽)
    public SlopeMaterial Labor { get; set; }      // Работа (м.п. × ₽/м.п.)
    
    // Производные
    public double TotalMaterials { get; }  // Σ позиций 1-7
    public double TotalLabor { get; }      // Позиция 8
    public double GrandTotal { get; }      // TotalMaterials + TotalLabor
    
    // Метаданные
    public int WindowCount { get; set; }   // Q (количество окон)
    public bool IsManualOverride { get; set; } // Были ли ручные правки
}

public class SlopeMaterial : INotifyPropertyChanged
{
    public string Name { get; set; }
    public double Quantity { get; set; }       // Расчётное кол-во
    public double ManualQuantity { get; set; } // Ручное (если отличается)
    public double Price { get; set; }
    public string Unit { get; set; }           // м², шт, м.п.
    public double Sum { get; }                 // Quantity × Price
    
    public bool IsQuantityOverridden { get; }  // ManualQuantity != Quantity
}
```

## 8. Пересчёт герметика/скотча по всему заказу

**Проблема:** При добавлении нового откоса или изменении Q существующего, все строки «Откос» должны пересчитать герметик и скотч (N_total изменился).

**Решение:**
- В `CalculationViewModel` — метод `RecalculateAllSlopes()`:
  1. Собирает все OrderItems где `SlopeData != null`
  2. Вычисляет `N_total = Σ SlopeData.WindowCount`
  3. Для каждого: пересчитывает `Sealant.Quantity = ceil(N_total / 4)`, `Tape.Quantity = ceil(N_total / 3)`
  4. Обновляет Total
- Вызывается при: добавлении/удалении строки «Откос», изменении Q
- Учитывает `IsManualOverride`: если пользователь вручную изменил кол-во герметика/скотча — авто-пересчёт не применяется к этой позиции

## 11. Открытые вопросы и риски

1. **Правая панель vs диалог:** Окончательное решение о UI выдвижной панели справа. Альтернатива: модальное окно (как предпросмотр КП). **Рекомендация:** выдвижная панель справа — не «засоряет» экран, переиспользует существующий паттерн, работает как просмотр детализации.

2. **Пересчёт герметика/скотча при ручных правках:** Если пользователь вручную изменил кол-во герметика в одном откосе, то при добавлении нового откоса — сбрасывать ли ручную правку? **Предложение в spec:** позиции с `IsManualOverride=true` не пересчитываются автоматически.

3. **Производительность при большом количестве откосов:** Пересчёт всех откосов при изменении одного — O(n). Для типичных заказов (1–10 окон) — не проблема.

4. **Совместимость со старыми заказами:** Старые заказы с «Откос материал» должны открываться. При сохранении — мигрировать? Или оставить «Откос материал» только для чтения старых заказов?
